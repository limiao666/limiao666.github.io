MVC

Posted on 2020-03-01 | In MVC

MVC设计模式:

M:model 模型:一个功能。用javabean实现

V:view视图 :用于展示，以及与用户交互。使用html,js,css,jsp,jquery等前端技术实现

C:controller控制器 :接受请求，将请求跳转到模型进行处理;模型处理完毕后，再将处理的返回给请求处. 可以用jsp实现，但是一般建议使用servlet实现控制器。

Jsp–>java(servlet)–>jsp

Servlet:

java类必须符合一定的规范:

必须继承 javax.servlet.http.HttpServlet

重写其中的doGet()或doPost()方法

  • doGet():接收并处理所有get提交方式的请求

  • doPost():接收并处理所有post提交方式的请求

Servlet要想使用，必须配置web.xml或者@WebServlet

项目的根目录: web content ,src

servlet流程:

请求–>url-pattern–>根据servlet-mapping中的servlet-name去匹配servlet中的servlet-name,然后寻找到servlet-class，然后将请求交给class

oracle安装配置

刚开始部署有点问题，搜索了半天也不知道怎么搞，最后决定移除oracle重新装。

  • 首先停止oracle: docker stop oracle

  • 移除oracle: docker rm oracle

  • 如果没有移除掉: dock rmi (image id)

  • 然后docker ps -a 和 docker images查看是否移除

  • 以下是重新部署步骤:

  • 下载: docker pull registry.cn-hangzhou.aliyuncs.com/helowin/oracle_11g

  • docker run -d -p 1521:1521 –name oracle registry.aliyuncs.com/helowin/oracle_11g

  • docker exec -it oracle bash

  • su root password为: helowin

  • vi /etc/profile

  • 将以下添加到profile最下面:

    export ORACLE_HOME=/home/oracle/app/oracle/product/11.2.0/dbhome_2

    export ORACLE_SID=helowin

    export PATH=$ORACLE_HOME/bin:$PATH

  • 保存退出后: source /etc/profile

  • su oracle

  • qlplus /nolog

  • su oracle

  • sqlplus /nolog

  • 进入sql以后: conn /as sysdba

  • alter user system identified by oracle;

  • alter user sys identified by oracle;

  • ALTER PROFILE DEFAULT LIMIT PASSWORD_LIFE_TIME UNLIMITED;

  • select * from user 执行这个语句应该会有信息。

  • 退出：两次exit 退到root层

  • 打开navicat创建oracle连接

  • ip为本机ip

  • Username：system

  • Password: oracle

hexo -d 报错:Spawn failed

Posted on 2020-02-25 | In hexo

在使用hexo和github搭建的博客时，hexo -d提交失败

经过google，stackoverflow………

网上的一种方法是直接修改.git下面的config文件

如果在使用hexo d命令的前提下，直接修改.git下面的config文件是无法成功的。原因是：使用该命令之后，会根据_config.yml下面的deploy节点进行cofig文件的覆盖。也就是说，_config.yml如果不做修改，无论如何修改.git下面的配置文件都是无效的，都会被覆盖。

终于找到了解决方法:

修改配置文件：根目录下的_config.yml，修改deploy节点。

deploy:

type: git

repo: https://github.com/{yourname}/{yourname}.github.io

branch: master

修改为如下：

deploy:

type: git

repo: https://{yourname}:{yourpassword}@github.com/{yourname}/{yourname}.github.io

branch: master

操作系统(一)

Posted on 2020-02-25 | In 操作系统

操作系统定义

  • 用户角度上，操作系统是一个控制软件

​ 管理应用程序，为应用程序提供服务，杀死应用程序，资源管理，管理外设，分配资源

  • 操作系统架层次结构

​ 硬件之上，应用程序之下

​ Linux,windows,android的界面属于外壳(shell),而不是内核(kernel),kernel是我们的研究重点，在shell之下。

Kernel–操作系统内部组件

  • 包括:(cpu,内存,磁盘)

  • cpu调度器，物理内存管理，虚拟内存管理，文件系统管理，中断处理与设备驱动

OS Kernel的特征:

1.并发:

  • 计算机系统中同时存在多个运行的程序，需要os管理和调度。并发是在一段时间内，有多个程序同时运行
  • 并行:一个时间点，要求存在多个cpu

2.共享:

  • “同时”共享

  • 互斥共享

3.虚拟化:

  • 比如把cup虚拟化成进程，把磁盘虚拟化成文件，把内存虚拟化成地址空间

  • 好处:使得我们每一个应用程序或者每一个用户感觉到自己在独自拥有一台计算机

  • 利用多道程序设计技术，让每个用户都觉得有一个计算机专门为他服务

4.异步：

  • 程序的执行部署一贯到底，而是走走停停，向前推进的速度不可预知

  • 但只要运行环境相同，os需要保证程序运行的结果也要相同

为什么要学习操作系统

综合许多不同的课程

  • 程序设计语言

  • 数据结构

  • 算法

  • 计算机体系结构

  • 材料

  • 操作系统概念和原理，源代码

  • 技能

  • 操作系统的设计与实现

操作系统还在不断地发展，在工控等各种领域有强烈的需求

操作系统很酷，很有用，有挑战

操作系统顶级会议：

  • SOSP(每两年)

  • USENIX(每两年)

操作系统需要权衡:

  • 空间与时间

  • 性能和可预测性

  • 公平和性能

Servlet - Servlet Applet - 小服务程序

用java语言编程的服务器端程序

如何实现自己的Servlet程序

  • 在线API - https://docs.oracle.com/javaee/7/api/

  • 所在的API - java.servlet.Servlet[i]

  • javax.servlet.Servlet接口定义一些方法，让我们自己写的servlet

    程序必须要实现这个接口中的所有方法

    • jdk8.0之前，接口只能定义抽象方法,jdk8及其以后的版本接口中

      ​ 是允许定义普通的方法，采取的是@default

  • servlet是用java编写的程序，运行在web服务器端

  • servlet可以用来接收和响应来自于web客户端(浏览器)的请求

    采取的协议是HTTP协议(超文本传输协议)

具体实现

  • 写一个类去实现javax.servlet.Servlet[i]这个接口

  • 写一个类去继承javax.servlet.GenericServlet[C] –支持通用协议的

  • 写一个类去继承javax.servlet.http.HttpServlet[C] – 仅仅支持http协议
