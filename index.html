IntelliJ IDEA打包war并部署运行

Posted on 2020-02-28 | In 服务器 , war

打包

打开idea-->File —> Project Structure —> Artifacts —> Add —> Web Application:Exploded-->From Modules--->选择项目-->ok--->然后再add-->Web Application:Archive-->选择上一步生成的war--->ok--->ok-->apply-->ok-->然后再到file一列，找到build--->build Artifacts-->选择:war-->build-->然后下面会显示正在生成war---->看到：build completer successfully--->打包成功！

部署运行:

  • 将war包复制到tomcat安装目录:/usr/local/apache-tomcat-8.5.51/webapps/下

  • 终端操作: cd /usr/local/apache-tomcat-8.5.51/bin/

  • 进入bin以后: ./startup.sh

  • 打开浏览器:http://ip:8080/xxx/xx.jsp

  • 部署以后 ./shutdown.sh关闭tomcat

安装步骤

  • 下载安装包

    官网下载地址：http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html

    我下载的是jdk-8u161-linux-x64.tar.gz

  • 解压安装包

  • 将我们下载好的JDK安装包上传到服务器，进行解压。进入目标文件夹中，我是放在/usr/local/java文件下，点击上传。

  • 解压: tar -zxvf jdk-8u161-linux-64.tar.gz

  • 移动jdk：mv jdk1.8.0_161 /usr/local/java/

  • 修改配置文件(配置环境变量): vim /etc/profile

  • 用vim编辑器来编辑profile文件，在文件末尾添加一下内容👇

    ：wq回车保存退出。

    保存之后通过source /etc/profile命令使其修改的配置生效

    测试jdk安装是否成功

    java -version

    下面开始安装tomcat

    • 安装地址:http://tomcat.apache.org/

    • 我下载的是apache-tomcat-8.5.51.tar.gz

    • 解压: tar -zxvf apache-tomcat-8.5.51.tar.gz

    • 我是放在了usr/local/tomcat下所以要创建tomcat文件夹并将解压文件移动到tomcat下

    • 移动tomcat: mv apache-tomcat-8.5.51 /usr/local/tomcat

    • 进入tomcat安装bin目录并启动:cd usr/local/tomcat/apache-tomcat-8.5.51/bin/

    • 进入bin目录以后：./startup.sh 会看到tomcat启动成功👇

    • 到浏览器输入: http://ip:8080

    • 如果无法连接，查看阿里云控制台，进入网络与安全-->安全组-->管理规则--->添加安全组规则--->将8080端口写进去，开放8080端口。

    • 之后再到浏览器输入: http://ip:8080 出现👇

    • 这样就安装成功啦！✌️

oracle安装配置

刚开始部署有点问题，搜索了半天也不知道怎么搞，最后决定移除oracle重新装。

  • 首先停止oracle: docker stop oracle

  • 移除oracle: docker rm oracle

  • 如果没有移除掉: dock rmi (image id)

  • 然后docker ps -a 和 docker images查看是否移除

  • 以下是重新部署步骤:

  • 下载: docker pull registry.cn-hangzhou.aliyuncs.com/helowin/oracle_11g

  • docker run -d -p 1521:1521 --name oracle registry.aliyuncs.com/helowin/oracle_11g

  • docker exec -it oracle bash

  • su root password为: helowin

  • vi /etc/profile

  • 将以下添加到profile最下面:

    export ORACLE_HOME=/home/oracle/app/oracle/product/11.2.0/dbhome_2

    export ORACLE_SID=helowin

    export PATH=$ORACLE_HOME/bin:$PATH

  • 保存退出后: source /etc/profile

  • su oracle

  • qlplus /nolog

  • su oracle

  • sqlplus /nolog

  • 进入sql以后: conn /as sysdba

  • alter user system identified by oracle;

  • alter user sys identified by oracle;

  • ALTER PROFILE DEFAULT LIMIT PASSWORD_LIFE_TIME UNLIMITED;

  • select * from user 执行这个语句应该会有信息。

  • 退出：两次exit 退到root层

  • 打开navicat创建oracle连接

  • ip为本机ip

  • Username：system

  • Password: oracle

Posted on 2020-02-22 | In Servlet

Servlet - Servlet Applet - 小服务程序

用java语言编程的服务器端程序

如何实现自己的Servlet程序

  • 在线API - https://docs.oracle.com/javaee/7/api/

  • 所在的API - java.servlet.Servlet[i]

  • javax.servlet.Servlet接口定义一些方法，让我们自己写的servlet

    程序必须要实现这个接口中的所有方法

    • jdk8.0之前，接口只能定义抽象方法,jdk8及其以后的版本接口中

      ​ 是允许定义普通的方法，采取的是@default

  • servlet是用java编写的程序，运行在web服务器端

  • servlet可以用来接收和响应来自于web客户端(浏览器)的请求

    采取的协议是HTTP协议(超文本传输协议)

具体实现

  • 写一个类去实现javax.servlet.Servlet[i]这个接口

  • 写一个类去继承javax.servlet.GenericServlet[C] --支持通用协议的

  • 写一个类去继承javax.servlet.http.HttpServlet[C] -- 仅仅支持http协议

    HelloServlet讲解的过程

    • 编码

      package tech.aistar.day01;

      //导包 //除了java.lang不需要手动导入，其余包中的类或者接口，都是需要导报 import javax.servlet.*; import java.io.IOException;

      /**

      • @Description 第一个servlet程序

      • @Author Miao Li

      • @Date 2020/2/22

      • @Version 1.0 / public class HelloServlet implements Servlet{ /*

        • 初始化方法
        • @param servletConfig
        • @throws ServletException */

        @Override public void init(ServletConfig servletConfig) throws ServletException { //随便在控制台输出一句话 - 主要目的是为了进行测试的 //看这个servlet程序是否启动成功，是否能够成功被执行

          //快速生成输出语句的快捷键 - sout
  System.out.println("init...");

        }

        /**

        • service方法 - 核心的
        • @param servletRequest
        • @param servletResponse
        • @throws ServletException
        • @throws IOException */ @Override public void service(ServletRequest servletRequest, ServletResponse servletResponse) throws ServletException, IOException { System.out.println("service..."); }

        /**

        • 销毁方法 */ @Override public void destroy() { System.out.println("destory..."); }

        /**

        • ServletConfig对象
        • @return */ @Override public ServletConfig getServletConfig() { return null; }


}

  • 编辑web.xml

    web3.0之前只有xml配置方法，学完xml配置方式才能更好理解

    注解开发的方式

    <?xml version="1.0" encoding="UTF-8"?> <web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_3_1.xsd" version="3.1">

    <!--传统的web3.0之前的配置方法，采用的是xml-->

    <!-- 每个servlet程序都是需要在xml中进行对应的配置-->

    <servlet>

    ​ <!--每个servlet标签都对应一个servlet程序，并且每个servlet程序在配置文件中-->

    ​ <!--都会拥有一个servlet-name,但是需要注意的是这个servlet-name在这个xml-->

    ​ <!--配置文件中必须要是唯一的，推荐采用servlet程序命名的小驼峰的命名方式，为了提高程序的可读性-->

    ​ <servlet-name>helloServlet</servlet-name>

    ​ <!--对应的servlet的类的全限定名 = 包名+类名

    ​ 检测 = ctrl按着不松+鼠标单击-->

    ​ <servlet-class>tech.aistar.day01.HelloServlet</servlet-class>

    </servlet>

    <!--每个servlet都有对应的一个映射路径-->

    <!--配置servlet的映射路径-->

    <servlet-mapping>

    ​ <!--此处的servlet-name一定是和某个servlet标签中的servlet-name是匹配的-->

    ​ <servlet-name>helloServlet</servlet-name>

    ​ <!-- url -统一资源定位器 -我们的servlet程序 -HelloServlet程序 -服务器中的-->

    <!--资源-->

    <!---->

    <!--url必须是唯一的，它是这个资源在服务器中的一定标志-->

    ​ <url-pattern>/day01/hello</url-pattern>

    </servlet-mapping>

    </web-app>

    • 打开浏览器 - 来访问servlet程序

      http://localhost:8080/servlet-demos/day01/hello

      http - 超文本传输协议

      Localhost - 服务器所在主机的ip地址，服务器是在本地(也可以写成127.0.0.1)

      8080 - tomcat配置的端口号，应用程序和端口号之间是1:1的关系。

      /servlet-demos - 项目的上下文路径 -暂时理解成某个servlet项目的项目名称

      /day01/hello - 某个servlet程序的资源路径，某个servlet的唯一标志。
